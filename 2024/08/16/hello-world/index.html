什么是TypeScript？

TypeScript 是 JavaScript 的类型的超集，⽀持 ES6 语法，⽀持⾯向对象编程的概念，如类、接⼝、继承、泛型等

超集:如果⼀个集合 A ⾥⾯的的所有元素集合 B ⾥⾯都存在，那么我们可以理解集合 B 是集合 A 的超集，集合 A 为集合 B 的⼦集

  • TypeScript其是⼀种静态类型检查的语⾔，提供了类型注解，在代码编译阶段就可以检查出数据类型的错误,同时扩展了 JavaScript 的语法，所以任何现有的 JavaScript 程序可以不加改变的在TypeScript 下⼯作;
  • 为了保证兼容性， TypeScript 在编译阶段需要编译器编译成纯 JavaScript 来运⾏，是为⼤型应⽤之开发⽽设计的语⾔;

基础数据类型

布尔类型 (boolean): 代表 true 或 false。

数字类型 (number): 代表整数和浮点数。

字符串类型 (string): 代表文本数据。

数组类型 (Array): 代表一组相同类型的值。

元组类型 (Tuple):

  • 元组是TypeScript中特有的类型，工作方式类似数组，最重要的特性是 可以限制数组元素的个数和类型，适合用来实现多值返回
  • 元组类型只能表示一个已知元素数量和类型的数组，长度已指定，越界访问会提示错误。 元组的解构赋值
  • 元组的解构赋值跟正常数组一样，但是要注意：解构数组元素的个数不能超过元组中元素的个数，不然会报错：

枚举类型 (enum): 代表一组命名的常量。

任何类型 (any): 代表任意类型，通常用于不确定类型的场景。

空类型 (void): 表示没有任何类型，通常用于函数没有返回值的情况。

未定义类型 (undefined): 表示变量未赋值。

空类型 (null): 表示一个空值。

联合类型（|）
联合类型是一种复合类型,它允许一个变量可以是多种类型中的任意一种。

联合类型使用场景

  • 1.函数参数: 当一个函数可以接受多种类型的参数时,可以使用联合类型来定义参数类型。
  • 2.返回值类型: 当一个函数可以返回多种类型的值时,可以使用联合类型来定义返回值类型。
  • 3.变量赋值: 当一个变量可以被赋予多种类型的值时,可以使用联合类型来定义变量类型。 类型别名( type )
